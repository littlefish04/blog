## 一、 博客撰写语法

### 1. Front Matter（元数据头部）

大多数博客系统（Hexo、Hugo、Jekyll 等）都使用 **Front Matter** 来定义文章属性：

```yaml
---
title: 文章标题
date: 2026-07-17 14:30:00
updated: 2026-07-17 15:00:00
tags:
  - Markdown
  - 博客
  - 教程
categories:
  - 技术分享
cover: /images/cover.jpg
description: 这是一篇关于如何写博客文章的介绍
---
```

把这样的一段内容放在 `.md` 文件的最开头就可以了。


**常用的 Front Matter 字段**：

| 字段 | 说明 | 是否必须 |
|------|------|---------|
| `title` | 文章标题 | ✅ 必须 |
| `date` | 创建日期 | ✅ 必须 |
| `updated` | 更新日期 | 可选 |
| `tags` | 标签（数组） | 可选 |
| `categories` | 分类（数组） | 可选 |
| `cover` | 封面图 | 可选 |
| `description` | 文章简介 | 可选 |
| `draft` | 是否为草稿（true/false） | 可选 |
| `author` | 作者 | 可选 |

### 2. Markdown 基础语法

> 本人是使用 `.md` 撰写博客的，故记录一下 Markdown 语法

#### (1) 标题

```markdown
# 一级标题（不推荐在正文中使用，留给文章标题）

## 二级标题（主要章节）

### 三级标题（子章节）

#### 四级标题（小节）

正文内容...
```

**建议**：
- 从 `##` 开始用，因为 `#` 通常和文章标题重复
- 层级不超过 4 级，保持清晰


#### (2) 段落和换行
```markdown
这是第一段内容。段落之间用空行分隔。

这是第二段内容。  
行尾加两个空格可以实现换行（不常用）。
```

#### (3)  强调
```markdown
*斜体* 或 _斜体_
**粗体** 或 __粗体__
***粗斜体*** 或 ___粗斜体___
~~删除线~~
`行内代码`
```
实际显示效果：  

*斜体*   
**粗体**  
***粗斜体***  
~~删除线~~  
`行内代码`


#### (4) 列表
```markdown
# 无序列表
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2

# 有序列表
1. 第一步
2. 第二步
   1. 子步骤2.1
   2. 子步骤2.2
3. 第三步
```
实际显示效果：  
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2

1. 第一步
2. 第二步
   1. 子步骤2.1
   2. 子步骤2.2
3. 第三步

#### (5) 链接
```markdown
# 链接
[显示文字](https://example.com "可选的标题")
```
实际显示效果是这样的：[显示文字](https://example.com "可选的标题")

#### (6) 引用
```markdown
> 这是一段引用内容
> 可以有多行
>> 还可以嵌套引用
```
实际效果：  
> 这是一段引用内容  
> 可以有多行
>> 还可以嵌套引用

#### (7) 代码块
````markdown
# 单行代码
`console.log('hello')`

# 多行代码（指定语言）
​```javascript
function hello() {
    console.log('Hello World!');
}
​```

# 多行代码（不指定语言）
​```
这是纯文本代码块
​```
````

#### (8) 表格
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```
实际显示效果：     

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |


#### (9) 分割线
```markdown
---
或
***
```

## 二、博客文章排版建议

### 最简单的文章结构模板（仅供参考）

```markdown
---
title: 文章标题（吸引人、包含关键词）
date: 2026-07-17
tags: [标签1, 标签2]
categories: 分类名称
---

## 标题1

### 要点1

...

## 标题2

...

## 参考资源（可选）

- [参考链接1](url)
- [参考链接2](url)
```

## 三、Hexo 自动生成标签插件

> Front Matter 里的 `date` 自己写起来太麻烦了，所以我去找了插件，自动生成岂不美哉

这类插件可以在你创建新文章时，根据文件名或当前时间，自动为你填充 Front-matter 中的 `date`、`title`，甚至 `categories` 和 `tags`。

### `hexo-auto-front-matter` 插件：

安装插件：在你的 Hexo 项目根目录下，运行安装命令：

```bash
npm install hexo-auto-front-matter --save

```

开启配置：在 Hexo 的主配置文件 `_config.yml` 中添加配置并开启它：

```yaml
# 开启 hexo-auto-front-matter 插件
auto_front_matter:
  enable: true
  
  # 自动根据文件名生成标题 (当标题为空时)
  auto_title:
    enable: true
    mode: 1
  
  # 自动用当前时间生成日期 (当日期为空时)
  auto_date:
    enable: true
  
  # 自动生成分类 (从文件目录结构)（未开启）
  auto_categories:
    enable: false
    depth: 1
    multiple: false
  
  # 自动生成标签（未开启）
  auto_tags:
    enable: false
  
  # 自动生成封面（未开启）
  auto_cover:
    enable: false
    per_category: {}
```

个人推荐把插件配置都写在文件末尾，你也可以选择其他地方。注意缩进别打错了。

配置好之后，你再使用 ``hexo new "文章标题"`` 创建新文件时，`date` 字段就会被自动填入当前时间，你就不用再手动去写了。

## 四、如何在博客里插入图片

在 Hexo 博客里加图片，主要有两种比较主流的方式。它们本质上的区别在于**你把图片放在哪儿**和**怎么引用它**。

### 方案一：使用统一的全局图片文件夹（最简单）

如果你的图片不多，或者想统一管理所有配图，这个方法是最直接、最省心的。

1. **存放图片**：把图片都放在 Hexo 项目根目录下的 `source/images/` 文件夹里（如果文件夹不存在就手动创建一个）。

2. **引用图片**：路径以 `/` 开头，指向站点根目录 `source` 。

    ```markdown
    ![图片描述文字](/images/你的图片文件.jpg)
    ```
    ![图片描述文字](/images/1.png)

**优点**是简单快捷。**缺点**是：

- 本地预览可能看不到图片（取决于你使用的 `.md` 编辑器），但是网页上显示是没问题的。


- 如果图片多了，都挤在一个文件夹里，时间长了会不太好管理。

### 方案二：开启“文章资源文件夹”功能（图片和文章一一对应，更推荐）

这个方法会让每篇文章都有自己的专属图片文件夹，管理起来会非常清晰，但配置上需要花一点点心思。

**第一步：开启功能**
在 Hexo 站点的主配置文件 `_config.yml` 里，找到并修改下面这行配置：
```yaml
post_asset_folder: true
```

**第二步：存放图片**
保存配置后，以后每次用 `hexo new "文章标题"` 新建文章时，Hexo 都会自动创建一个和文章同名的文件夹。把这篇博文需要用到的所有图片都放进这个文件夹里就行了。

**第三步：引用图片**
因为图片是跟着文章的，引用方式会稍微复杂些，**通常有三种主流方法**，它们的兼容性和预览效果略有不同：

**方法A：使用 Hexo 专属标签插件**
这是 Hexo 官方文档里明确推荐的方式，它能保证在首页和文章页都能正常显示。

```markdown
<!-- 标签插件语法，会解析成 <img> 标签 -->
{% asset_img 你的图片文件名.jpg 图片描述文字 %}
```
*需要留意的是：这种方式在 VS Code 等编辑器里预览 Markdown 时可能看不到图片，但部署到博客上是没问题的。*

{% asset_img 1.png 图片描述文字 %}

**方法B：配置新版 Markdown 渲染器**
从 `hexo-renderer-marked` 的 3.1.0 版本开始，Hexo 支持用标准的 Markdown 语法写相对路径，渲染时会自动处理。

可以在 Hexo 站点根目录下，在终端里输入 `npm list --depth=0` 来查看当前 `hexo-renderer-marked` 的版本。笔者写这篇文章的时候都已经到 7.0.1 版本了...

如果你用的渲染器版本足够新，可以这样配置 `_config.yml`：

```yaml
post_asset_folder: true
marked:
  prependRoot: true
  postAsset: true
```
配置好后，在文章里就可以直接用标准的 Markdown 语法了：
```markdown
![图片描述文字](你的图片文件名.jpg)
```

![图片描述文字](1.png)

这种方式在本地编辑器里预览 Markdown 时，不做单独设置的话仍然看不到图片，但部署到博客上是没问题的。

*如果你使用的编辑器是Typora：* 可以在 编辑 - 图片工具 - 设置图片根目录... 里把根目录设为上面提到的那个文章同名文件夹。这样你就能在本地编辑器里看到图片了。



**方法C：借助社区插件（仅在方法B使用不了的时候推荐）**
如果觉得手动配置比较繁琐，也可以安装社区插件来辅助，比如 `hexo-asset-image` 或 `hexo-compatible-image`。这些插件的目标是让你直接使用 `![](图片名.jpg)` 这类相对路径，在本地预览和网页显示时都能正常工作。不过，部分较老的插件在安装后可能需要对源码做微调才能完美适配。
