## 写在前面

虽然现在博客建好了，页面看起来也挺像样，甚至还有评论系统，但是！在搜索引擎根本搜不到博客网站啊！只能靠手动分享链接才能让别人看到自己的博客。

笔者于是乎去搜集了让博客能被搜索引擎收录的方法，还有一些相关的插件。以下内容以 Google, Bing 两个搜索引擎为例，其它搜索引擎的步骤也是类似的。

部分内容为 ai 生成，如有错误恳请指出。

## 一、配置优化

### 文章永久链接 — hexo-abbrlink

> 插件仓库：<https://github.com/rozbo/hexo-abbrlink>

有没有遇到过这种情况：文章发布后想改个更合适的标题，结果链接跟着变了，之前分享出去的链接全变成 404？

`hexo-abbrlink` 解决的就是这个问题。它会为每篇文章生成一个唯一的、固定不变的 ID，用这个 ID 替代标题作为永久链接。之后不管你改多少次标题、挪到什么分类下，链接都不会变。

生成后，每篇文章的 Front-matter 中会自动添加 `abbrlink` 字段，值就是生成的 ID，这个字段确保了修改文件名或标题后链接地址保持不变。

在项目根目录下运行以下命令安装插件：

```bash
npm install hexo-abbrlink --save
```

在 `_config.yml` 中修改永久链接配置：

```yaml
permalink: posts/:abbrlink/

abbrlink:
    alg: crc32   # 算法：crc16（默认）和 crc32
    rep: dec     # dec 数字 / hex 哈希串
```

`alg` 和 `rep` 决定了 ID 的生成形式，不同组合效果如下：

| 算法 | 格式 | 示例 |
|------|------|------|
| crc16 + hex | 短哈希 | `posts/66c8.html` |
| crc16 + dec | 短数字 | `posts/65535.html` |
| crc32 + hex | 长哈希 | `posts/8ddf18fb.html` |
| crc32 + dec | 长数字 | `posts/1690090958.html` |

### Head 头部优化

搜索引擎会读取页面的 `<meta>` 标签来理解网页内容。`_config.yml` 中的 `keywords`、`description` 等字段会被注入到每个页面的 `<head>` 中。

我的站点级配置（`_config.yml`）：

```yaml
# Site
title: littlefish04个人博客
subtitle: ''
description: '记录一些学习过程'
keywords: 博客,littlefish04 blog,littlefish04个人博客
author: littlefish04
language: zh-cn
```

主题配置文件 `_config.Anatolo.yml` 也会覆盖部分字段：

```yaml
keywords: Blog,博客,Hexo,littlefish04 blog,littlefish04个人博客
author: littlefish04
description: '记录一些学习过程'
```

> 注：`keywords` 在现代搜索引擎中的权重已经大不如前，但建议仍然填写，用逗号分隔关键词即可。`description` 更为重要——它会直接出现在搜索结果摘要中。

### 外链优化 — hexo-filter-nofollow

> 插件仓库：<https://github.com/hexojs/hexo-filter-nofollow>

博客里难免会引用外部链接——参考文章、官方文档等。但这些外链会将页面的权重传递出去，不利于 SEO。

`hexo-filter-nofollow` 会为所有外部链接自动添加 `rel=”noopener external nofollow noreferrer”` 属性，告诉搜索引擎”这个链接跟我没关系，别把权重分给它”，同时也能防止安全风险。

安装：

```bash
npm install hexo-filter-nofollow --save
```

在 `_config.yml` 中开启：

```yaml
nofollow:
  enable: true
  field: site
  exclude:
    - 'exclude1.com'
    - 'exclude2.com'
```

`exclude` 列表里可以放你信任的域名，这些域名的外链不会被添加 nofollow 属性。

## 二、站点地图与爬虫指引

上面那些配置只是优化了页面本身的 SEO 标签。但要被搜索引擎收录，还得主动告诉它：我的网站有哪些页面、哪些该爬、哪些不用。

### 1. Sitemap — hexo-generator-sitemap

> 插件仓库：<https://github.com/hexojs/hexo-generator-sitemap>

Sitemap（站点地图）是一个 XML 文件，列出你网站上所有页面的地址、更新频率、优先级。搜索引擎爬虫在抓取新网站时，第一件事就是找 sitemap——它相当于给了搜索引擎一张"网站导航图"。

安装：

```bash
npm install hexo-generator-sitemap --save
```

在 `_config.yml` 中添加配置：

```yaml
# SEO - Sitemap
sitemap:
  path:
    - sitemap.xml
    - sitemap-articles.xml
  rel: true
  tags: true
  categories: true
```

配置说明：
- `path`：生成两个 sitemap 文件。`sitemap.xml` 包含所有页面，`sitemap-articles.xml` 只包含文章
- `rel: true`：在页面 `<head>` 中添加 sitemap 关联标签
- `tags: true` / `categories: true`：把标签页和分类页也纳入站点地图

构建后，`sitemap.xml` 会生成在 `public/` 目录下，部署后可以通过 `https://你的域名/sitemap.xml` 访问。

### 2. Robots.txt — hexo-generator-robotstxt

> 插件仓库：<https://github.com/hexojs/hexo-generator-robotstxt>

`robots.txt` 是放在网站根目录下的一个纯文本文件，告诉搜索引擎爬虫：哪些目录可以抓、哪些不行。比如 CSS 和 JS 文件夹就没必要让爬虫收录。

安装：

```bash
npm install hexo-generator-robotstxt --save
```

在 `_config.yml` 中添加配置：

```yaml
# SEO - Robots.txt
robotstxt:
  user_agent: '*'
  allow:
    - /
  disallow:
    - /css/
    - /js_complied/
    - /js/
  sitemap: https://littlefish04.github.io/blog/sitemap.xml
```

`user_agent: '*'` 表示对所有爬虫生效（Googlebot、Bingbot 等都适用）。最后一行指向 sitemap 的完整 URL，方便爬虫直接发现。

### 3. 修复 URL 协议

这里还有一个容易被忽视的小问题：`_config.yml` 中的 `url` 字段一开始是 `http://`，但 GitHub Pages 强制使用 HTTPS。如果不改成 `https://`，会导致 sitemap 中的所有链接都是 `http://` 开头，搜索引擎就得先经历一次重定向。

```yaml
url: https://littlefish04.github.io/blog
```

## 三、社交分享预览 — Open Graph 与 Twitter Card

你在微信、Telegram、Twitter 上分享链接时有没有见过这种情况：有的链接会带标题和缩略图卡片，有的就只是一个干巴巴的蓝色 URL？

差别就在于有没有 Open Graph（`og:`）标签——这些标签会在页面 `<head>` 中告诉社交平台"请用这个标题、这个描述、这张图作为预览卡片"。

### 修改 head.pug

在 `themes/主题名/layout/partial/head.pug` 中，ATOM feed 链接之后添加：

```pug
//- Open Graph（Facebook、LinkedIn、Telegram 等社交平台分享预览）
- var og_title = page.title ? page.title + ' · ' + config.title : config.title
- var og_desc = page.description || page.desc || theme.description || config.description
- var og_url = config.url + '/' + (page.path || '')
- var og_img = page.cover || page.image || config.url + '/images/1.png'
meta(property="og:title", content=og_title)
meta(property="og:type", content=page.layout === 'post' ? 'article' : 'website')
meta(property="og:url", content=og_url)
meta(property="og:description", content=og_desc)
meta(property="og:image", content=og_img)
meta(property="og:site_name", content=config.title)
meta(property="og:locale", content=config.language === 'zh-cn' ? 'zh_CN' : 'en_US')
if page.layout === 'post'
  if page.date
    meta(property="article:published_time", content=page.date.toISOString())
  if page.updated
    meta(property="article:modified_time", content=page.updated.toISOString())
  each tag in (page.tags ? page.tags.toArray() : [])
    meta(property="article:tag", content=tag.name)

//- Twitter Card
meta(name="twitter:card", content="summary")
meta(name="twitter:title", content=og_title)
meta(name="twitter:description", content=og_desc)

//- Canonical URL（防止重复内容被搜索引擎降权）
link(rel="canonical", href=og_url)
```

各标签的作用：

| 标签 | 作用 |
|------|------|
| `og:title` | 分享卡片的标题 |
| `og:description` | 分享卡片的描述文字 |
| `og:image` | 分享卡片的缩略图 |
| `og:url` | 页面的规范链接 |
| `og:type` | 页面类型：文章为 `article`，首页为 `website` |
| `article:published_time` | 文章发布日期（仅文章页） |
| `twitter:card` | Twitter/X 分享时的卡片样式 |
| `canonical` | 告诉搜索引擎"这个地址是唯一官方地址"，防止重复内容被降权 |

## 四、提交到搜索引擎

做完以上配置并部署后，搜索引擎还不会立刻知道你的博客存在。最后一步——主动把网站提交给 Google 和 Bing。

### Google Search Console

1. 打开 [Google Search Console](https://search.google.com/search-console/)，用 Google 账号登录

2. 添加资源 → 选择**网址前缀 (URL prefix)** → 输入你的博客的 url，比如笔者的就是 `https://littlefish04.github.io/blog` 

3. 验证所有权。推荐选择 **HTML 标签 (HTML tag) **方式，Google 会给一段类似下面的代码：
   ```html
   <meta name="google-site-verification" content="一串随机字符" />
   ```
   把这段代码加到 `themes/主题名/layout/partial/head.pug ` 中,

   在第 1 行和第 2 行之间插入，像这样：

   ```pug
   meta(charset="utf-8")
   meta(name="google-site-verification", content="Google给的那串随机字符")    // ← 加在这里
   meta(name="X-UA-Compatible", content="IE=edge")
   ```

   为什么加在这个位置？ <head> 中的 meta 标签没有严格的顺序要求，但紧跟在 charset    后面是最清晰、最不容易被遗漏的位置。以后如果还要加 Bing 验证标签或者百度验证标签，也可以一并堆在这里。

   然后部署网站并验证。

4. 验证成功后，左侧菜单 → **Sitemaps** → 提交 `sitemap.xml`

5. 之后可以在这里查看收录状态、搜索点击量等统计

### Bing Webmaster Tools

1. 打开 [Bing Webmaster Tools](https://www.bing.com/webmasters/)，用 Microsoft 账号登录
2. 推荐直接**从 Google Search Console 导入**——Bing 会读取你已验证的站点，一键完成
3. 如果选择手动添加，验证方式同样是 HTML meta 标签（和 Google 类似）
4. 提交 sitemap 后即可在仪表板监控收录情况

> 搜索引擎收录新站点需要时间，从几小时到几周都有可能。耐心等待，定期检查 Search Console 中的索引报告即可。
